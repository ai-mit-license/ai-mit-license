# Translation Guide

Thank you for helping translate the AI-MIT License.

## Important rules

1. **Translations are non-authoritative.** The English text is always the legally binding version. Every translation file must include this notice at the top:
   ```
   > This translation is for informational purposes only and has no legal force.
   > The authoritative text is the English original: ../LICENSE
   ```

2. **Translate meaning, not words.** Legal language is precise. Prioritize conveying the correct legal meaning in the target language over literal word-for-word translation.

3. **Keep technical terms consistent.** The following terms should be translated consistently throughout the document:
   | English | Notes |
   |---|---|
   | Authorship Declaration | The checkbox block at the top |
   | Fully AI-generated | Mode 1 |
   | AI-assisted | Mode 2 |
   | Human-authored | Mode 3 |
   | Originator | The person/org distributing the code |
   | Software | The licensed work |
   | public domain | Use local legal equivalent where applicable |

4. **Do not change the structure.** Keep the same sections in the same order. Do not add or remove clauses.

5. **Note jurisdiction-specific issues.** If a term (e.g., "public domain") does not exist or works differently in the target jurisdiction, add a translator's note in *italics* explaining the difference.

## File naming

Name your file using the ISO 639-1 language code:
- `de.md` — German
- `fr.md` — French
- `zh.md` — Chinese (Simplified)
- `zh-tw.md` — Chinese (Traditional)
- `ja.md` — Japanese
- `es.md` — Spanish
- `pt.md` — Portuguese
- `ru.md` — Russian (already exists)

## Submitting a translation

Open a Pull Request with your translation file in `/translations/`. In the PR description, include:
- Your language and locale
- Any legal notes about jurisdiction-specific differences
- Whether you are a legal professional (optional but helpful)
- Whether the translation was AI-assisted (note which tool)
